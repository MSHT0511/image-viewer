# Image Viewer - E2E Testing with Playwright

## テストの実行方法

### 前提条件
- アプリケーションがビルドされている必要があります
- Playwrightがインストールされている必要があります（npm installで自動インストール）

### テストコマンド

```bash
# 基本的なテスト実行（ヘッドレスモード）
npm test

# UIモードでテスト実行（インタラクティブ）
npm run test:ui

# ブラウザを表示してテスト実行
npm run test:headed

# デバッグモードでテスト実行
npm run test:debug

# アプリのビルドのみ
npm run test:build
```

## テスト項目

### 1. アプリの基本表示 (`e2e/app.spec.ts`)
- ✅ アプリウィンドウの表示
- ✅ ツールバーの表示と機能ボタン
- ✅ ズームコントロールの表示
- ✅ ナビゲーションコントロールの表示
- ✅ ファイルドロップゾーンの表示
- ✅ 画像ビューワーの表示
- ✅ ズームボタンの動作確認
- ✅ 初期状態のスクリーンショット撮影

### 2. 画像読み込みテスト (`e2e/image-loading.spec.ts`)
- ✅ 画像未読み込み時のプレースホルダー表示
- ✅ キーボードショートカットの動作確認
- ✅ ツールバーのファイル数表示
- ✅ UIレスポンシブネステスト
- ✅ 連続したズーム操作のテスト
- ✅ エラーハンドリング確認

## テスト構成

```
e2e/
├── electron-app.ts          # Electronアプリ起動ヘルパー
├── app.spec.ts              # 基本的なUI表示テスト
├── image-loading.spec.ts    # 画像読み込みテスト
└── screenshots/             # テスト実行時のスクリーンショット
    └── .gitignore
```

## テスト結果

テスト実行後、以下が生成されます：

1. **HTMLレポート**: `playwright-report/index.html`
   - ブラウザで開いて詳細な結果を確認できます
   - 失敗時のスクリーンショットも含まれます

2. **スクリーンショット**: `e2e/screenshots/`
   - テスト中に撮影されたスクリーンショット
   - 失敗時の画像

3. **トレース**: 失敗したテストのトレースファイル
   - リトライ時に記録されます

## トラブルシューティング

### テストが失敗する場合

1. **アプリがビルドされていない**
   ```bash
   npm run test:build
   ```

2. **ポートが使用中**
   - 他のElectronインスタンスを終了してください

3. **タイムアウトエラー**
   - `playwright.config.ts`の`timeout`を増やしてください

### デバッグ方法

```bash
# Playwrightインスペクタでステップ実行
npm run test:debug

# UIモードでインタラクティブにテスト
npm run test:ui

# ヘッドフルモードで実行（ブラウザ表示）
npm run test:headed
```
